# Phase 7 Checkpoint

**Status:** Native implementation candidate verified, independently approved, and reorganized as the primary project.
**Date:** 2026-05-28.
**Scope of this checkpoint:** macOS SwiftUI native app, Supabase native RPC boundary, EventKit write-through adapter, retry visibility, parity documents, release-gate evidence, and native-primary repo layout.

## Implementation Produced

| Area | Outcome |
|------|---------|
| Native project | `DrumLessonOS/`, `DrumLessonOSTests/`, `project.yml`, and `DrumLessonOS.xcodeproj` contain a macOS SwiftUI app generated by XcodeGen and verified through `xcodebuild`. |
| Domain/read models | Swift value models, read-model mapping, weekly occurrence expansion, scheduling validation, and student editing validation mirror the web contract. |
| Supabase live boundary | Native auth/session, REST reads, authenticated RPC writes, and publishable-key validation are implemented without service-role embedding. |
| Student workflows | Native roster, detail, profile editing, traits, progress, assignments, notes, next plan, lesson flow, and closeout workflows are implemented. |
| Scheduling workflows | One-off create, weekly schedule expansion, occurrence edit/cancel, start lesson, and native sync status display are implemented. |
| EventKit adapter | Calendar permission state, writable calendar selection, create/update/delete, event-note ownership marker, and failure capture are implemented. |
| Retry behavior | EventKit failures are queued with the occurrence id and can be retried without duplicating Supabase occurrences. |
| Legacy web runtime | Removed from active development after Phase 7 approval so the repo opens as a native macOS app. |

## Known External Gates

These are not marked complete by this checkpoint because they require live credentials, a real Apple Calendar permission prompt, or an iPhone signed into the same iCloud account:

- Real instructor sign-in with production Supabase credentials.
- Real EventKit permission prompt, denial/revocation recovery, and writable calendar selection.
- Mac Apple Calendar create/edit/cancel proof using a real selected calendar.
- iPhone iCloud propagation proof.
- One-week daily-use fallback window before retiring the web app.

## Release Position

Phase 7 is code-complete as a native implementation candidate and has been reorganized as the project root. Live use remains gated by the UAT items above.
